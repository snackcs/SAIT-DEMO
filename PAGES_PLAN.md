# PAGES_PLAN — Детальный план каждой страницы

---

## Философия контента демо-страниц

### Главное: для кого эти страницы

Каждая демо-страница — это не портфолио фрилансера и не абстрактный шаблон. Это **полноценный сайт конкретного бизнеса**, который показывается потенциальному клиенту из этой ниши. Барбер смотрит `/barbershop` и должен думать: "Хочу именно такой сайт для себя." Владелец автосервиса смотрит `/auto` и узнаёт свой бизнес в контенте.

Цель каждой демо-страницы — **продать сайт фрилансера через качество самого демо**.

---

### 1. Цель каждой демо-страницы

Пользователь (потенциальный клиент фрилансера), попадая на страницу, должен **немедленно понять**:

- что это за бизнес и чем он занимается
- какие услуги предоставляет и сколько они стоят
- в чём преимущества этой компании перед конкурентами
- кто выполняет работу (мастер, команда, специалист)
- почему этой компании можно доверять (отзывы, кейсы, факты)
- как записаться, позвонить, написать
- где находится бизнес или как с ним связаться
- что именно можно адаптировать под его собственный бизнес (через DemoBanner)

---

### 2. Принцип "без белых пятен"

На каждой странице **не должно быть смысловых пустот**. Каждый ключевой вопрос клиента закрывается отдельным блоком или понятным элементом интерфейса.

Страница обязана закрывать все эти вопросы:

| Вопрос клиента                              | Блок, который отвечает          |
|---------------------------------------------|---------------------------------|
| Что это за компания?                        | Hero — название, бейдж, оффер  |
| Чем занимается?                             | Hero — подзаголовок             |
| Какие услуги и цены?                        | Services + Pricing              |
| Можно ли доверять?                          | Reviews + Gallery               |
| Кто делает работу?                          | About/мастер в Hero или секции  |
| Почему выбрать их, а не конкурентов?        | WhyUs / преимущества в Hero     |
| Как происходит запись / обращение?          | Contacts + CTA + процесс       |
| Где находятся / как связаться?              | Contacts                        |
| Есть ли примеры работ?                      | Gallery / Results               |
| Что делать прямо сейчас?                    | NicheCTA                        |

---

### 3. Пользовательский путь (сценарий просмотра)

Каждая страница выстроена по логике продающего пути. Последовательность секций не случайна:

```
1. Привлечь внимание       → Hero (сильный оффер, визуал)
2. Объяснить суть          → Services (что конкретно предлагается)
3. Показать доказательства → Gallery / Results (примеры работ)
4. Снять сомнения          → Reviews (реальные отзывы)
5. Обосновать цену         → Pricing (прозрачные тарифы)
6. Подтолкнуть к действию  → NicheCTA (финальный призыв)
7. Дать способ связаться   → Contacts
```

Страница ведёт клиента сверху вниз, и на каждом шаге усиливает желание заказать.

---

### 4. Лимиты символов для контентных полей

Без лимитов разработчик заполнит текст произвольной длины → карточки разной высоты, сломанная сетка, кривой layout. Лимиты обязательны.

| Поле                              | Лимит          | Комментарий                                      |
|-----------------------------------|----------------|--------------------------------------------------|
| `service.title`                   | ≤ 35 символов  | Одна строка в карточке услуги                    |
| `service.description`             | 70–110 символов| 2 строки, одинаковая высота карточек             |
| `service.price`                   | ≤ 20 символов  | "от 800 ₽" / "по запросу"                        |
| `review.text`                     | 80–140 символов| 3–4 строки, карточки отзывов не прыгают          |
| `review.name`                     | ≤ 20 символов  | "Алексей М." — имя + инициал                     |
| `review.role`                     | ≤ 25 символов  | "клиент" / "BMW 3 Series" / "мама ученика"       |
| `pricingPlan.name`                | ≤ 20 символов  | "Базовый" / "Стандарт" / "VIP"                   |
| `pricingPlan.features[]` (каждый) | ≤ 45 символов  | Один пункт списка тарифа — одна строка           |
| `pricingPlan.features.length`     | 3–5 пунктов    | Меньше — скудно, больше — не помещается           |
| `about.text`                      | 200–300 символов | 3–4 предложения. Не роман, не слоган            |
| `about.facts[].label`             | ≤ 15 символов  | "Мастеров" / "Лет работы" / "Клиентов"           |
| `about.facts[].value`             | ≤ 10 символов  | "6" / "7 лет" / "2000+"                          |
| `whyUs[].title`                   | ≤ 30 символов  | Заголовок преимущества                           |
| `whyUs[].text`                    | 60–90 символов | Расшифровка — 2 строки                           |
| `process[].title`                 | ≤ 25 символов  | "Запись" / "Диагностика" / "Результат"           |
| `process[].text`                  | 60–90 символов | Что происходит на этом шаге                      |
| `faq[].question`                  | ≤ 60 символов  | Умещается в аккордеоне одной строкой             |
| `faq[].answer`                    | 100–200 символов | Раскрытый ответ, но не статья                  |
| `hero.tagline`                    | ≤ 80 символов  | Подзаголовок Hero — 2 строки на мобиле           |
| `seo.description`                 | 120–160 символов | Google обрезает после 160                      |

---

### 5. Обязательные смысловые блоки каждой демо-страницы

Недостаточно просто показать красивый дизайн. Каждая страница должна быть наполнена **информационно**:

- **Hero с сильным оффером** — конкретный заголовок (не "Лучший сервис в городе", а "Стрижки в Москве от 800 ₽")
- **Краткое описание бизнеса** — кто, сколько лет, сколько клиентов (бейдж в Hero или отдельный блок)
- **Список услуг** — с иконками, описанием и ценами "от"
- **Преимущества** — коротко, конкретно, без воды
- **Как проходит работа / запись** — 3 шага, снижает тревогу клиента
- **Цены и пакеты** — прозрачность = доверие
- **Отзывы** — с именами, ролями, конкретикой
- **Примеры работ / кейсы** — галерея или блок результатов
- **FAQ** — ответы на типовые вопросы ниши (у каждой — свои)
- **Контакты** — адрес, телефон, мессенджеры, часы работы
- **Форма записи или CTA** — финальное действие
- **DemoBanner** — "Это демо. Адаптируем под ваш бизнес."

---

### 6. Адаптация под логику ниши

Наполненность — это не "напихать всего подряд". Контент, заголовки, CTA, FAQ и акценты должны **соответствовать ожиданиям целевой аудитории конкретной сферы**:

| Ниша         | Главные акценты в контенте                                          |
|--------------|---------------------------------------------------------------------|
| Барбершоп    | Атмосфера, мастера, стиль, фото работ, онлайн-запись               |
| Автосервис   | Надёжность, гарантия, быстрота, марки авто, прайс на ТО            |
| Стоматология | Доверие, безболезненность, врачи, оборудование, до/после           |
| Салон красоты| Красота результата, мастерство, уютная атмосфера, онлайн-запись     |
| Репетитор    | Результаты учеников (баллы), методика, форматы, первый урок бесплатно |
| Кофейня      | Атмосфера, меню с ценами, фото, адрес, часы работы                 |

---

### 7. Требования к текстам

Тексты должны быть **конкретными, живыми и понятными** — как на настоящем коммерческом сайте:

- **Нельзя**: "Мы профессиональная команда с большим опытом работы"
- **Можно**: "8 мастеров. Работаем с 2017 года. Более 5000 стрижек."

- **Нельзя**: "Качественные услуги по доступным ценам"
- **Можно**: "Классическая стрижка — от 800 ₽. Запись за 2 минуты."

- **Нельзя**: "Мы всегда рады помочь вам"
- **Можно**: "Напишите в Telegram — ответим в течение 15 минут"

Каждый блок **передаёт полезный смысл**, а не просто занимает место на странице.

---

### 8. FAQ — индивидуальный для каждой ниши

FAQ — не украшение, а инструмент снятия возражений. Вопросы должны быть **реальными для этой сферы**:

| Ниша         | Примеры вопросов FAQ                                                 |
|--------------|----------------------------------------------------------------------|
| Барбершоп    | Нужна ли предварительная запись? Работаете с детьми? Сколько длится стрижка? |
| Автосервис   | Есть ли гарантия на работы? Принимаете иномарки? Сколько ждать?     |
| Стоматология | Больно ли? Как проходит первый приём? Принимаете по полису?         |
| Салон красоты| Можно прийти без записи? Используете профессиональные материалы?    |
| Репетитор    | Онлайн или офлайн? Как проходит первый урок? Гарантируете результат? |
| Кофейня      | Можно ли работать за ноутбуком? Есть ли веганское меню? Бронирование столов? |

## Главная страница — /

### Цель
Витрина фрилансера. Посетитель видит кто ты, что делаешь, примеры работ по нишам и оставляет заявку.

### Секции

#### 1. Header (глобальный)
- Логотип / имя слева
- Навигация по центру: Главная · Примеры · Контакты
- Кнопка справа: [Заказать сайт] → /contact
- На мобиле: бургер-меню, навигация выезжает снизу
- При скролле: фон хедера становится непрозрачным (backdrop-blur)

#### 2. Hero
- Полноэкранная секция (min-h: 100vh)
- Левая колонка: заголовок + подзаголовок + кнопки
- Правая колонка: мозаика из 6 превью ниш (анимированная сетка)
- Заголовок: "Сайты для бизнеса под ключ"
- Подзаголовок: "Современные лендинги и многостраничники для вашей ниши. Быстро. С гарантией результата."
- Кнопки: [Смотреть примеры] (скролл вниз) · [Обсудить проект →] (/contact)
- Анимация: заголовок появляется снизу, кнопки с задержкой
- Мобиле: колонки в стопку, мозаика — горизонтальный скролл

#### 3. NicheGrid — "Выбери свою нишу"
- Заголовок секции: "Примеры сайтов по нишам"
- Подзаголовок: "Нажми на любую карточку — откроется полноценное демо"
- Сетка 3×2 карточек NicheCard
- Каждая карточка:
  - Фоновый цвет ниши (полупрозрачный)
  - Иконка ниши (lucide-react)
  - Название: "Барбершоп", "Автосервис" и т.д.
  - Короткий тег: "Лендинг · Запись онлайн"
  - При hover: карточка поднимается, появляется кнопка "Смотреть демо →"
  - Клик → переход на страницу ниши
- Мобиле: сетка 2×3 или 1×6

#### 4. WhyMe — "Почему я"
- Заголовок: "Что ты получаешь"
- 4 блока в ряд с иконкой и текстом:
  - Скорость: "Сайт за 5–7 дней, не за месяцы"
  - Адаптив: "Корректно на любом устройстве"
  - Современный дизайн: "Не шаблонный WordPress, а живой дизайн"
  - Поддержка: "Помогу после запуска, не исчезну"
- Мобиле: 2×2 или стопка

#### 5. HowItWorks — "Как это работает"
- Заголовок: "3 шага до готового сайта"
- 3 блока с номером, иконкой и текстом:
  1. Оставь заявку — заполни форму или напиши в Telegram
  2. Обсудим детали — расскажи про бизнес, я предложу структуру
  3. Получи сайт — финальный результат с правками включены
- Между блоками стрелки (на десктопе)
- Мобиле: вертикальный список с линией-разделителем

#### 6. HomeCTA — финальный блок
- Тёмный фон с акцентным градиентом
- Заголовок: "Готов обсудить твой проект"
- Подзаголовок: "Напиши — отвечу в течение часа"
- Кнопка: [Оставить заявку] → /contact
- Под кнопкой: иконки Telegram + Email

#### 7. Footer (глобальный)
- Логотип + короткое описание слева
- Ссылки на ниши по центру
- Контакты справа: Telegram, Email
- Копирайт внизу

## SEO — generateMetadata() для каждой страницы

Каждая страница ниши экспортирует `generateMetadata()`:

```ts
// пример для /barbershop
export const metadata: Metadata = {
  title: 'Сайт для барбершопа — демо-шаблон | Илья Фрилансер',
  description: 'Пример сайта для барбершопа. Современный дизайн, онлайн-запись, адаптив. Под ваш бизнес за 5 дней.',
  openGraph: {
    title: 'Сайт для барбершопа — смотри демо',
    description: 'Современный лендинг для барбершопа. Адаптируем под ваш бренд.',
    images: [{ url: 'https://images.unsplash.com/...' }],
  },
}
```

Главная страница:
```ts
title: 'Сайты для бизнеса под ключ | Илья Фрилансер',
description: 'Делаю сайты для барбершопов, автосервисов, салонов красоты, стоматологий, кофеен и репетиторов.',
```

## Служебные страницы

### not-found.tsx (кастомная 404)
- Заголовок: "Страница не найдена"
- Подзаголовок: "Возможно, ссылка устарела или в ней опечатка"
- Кнопка: [На главную]
- Кнопка: [Написать в Telegram] → прямая ссылка
- В едином стиле сайта, не дефолтный Next.js экран

### loading.tsx (глобальный)
- Пульсирующий skeleton или простой спиннер
- Используется Next.js автоматически при навигации

### error.tsx (глобальный)
- Заголовок: "Что-то пошло не так"
- Кнопка: [Попробовать снова] — вызывает `reset()`
- Кнопка: [На главную]
